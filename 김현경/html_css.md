## DOCTYPE에 대하여 설명하시오

- Document Type의 약자로 HTML 문서의 유형을 정의하기 위해 사용하는 선언문
- HTML이 어떤 버전으로 작성되었는지 미리 선언해서 웹 브라우저가 내용을 올바르게 표시할 수 있도록 해주는 것
- DOCTYPE이 없다면 웹 브라우저가 문서를 잘못 해석하거나 렌더링하는 경우 발생

## meta 태그에 대해서 알고 있나요?

- `meta` 태그는 HTML 문서에서 메타데이터(문서 정보를 설명하는 데이터)를 정의하기 위해 사용하는 태그
- 웹 브라우저와 검색 엔진이 문서의 정보를 더 잘 이해할 수 있도록 도와줌

## meta 태그의 요소에 대해서 아는대로 말해보세요

### `charset`

- 문서의 문자 인코딩 방식을 지정
- UTF-8은 다양한 언어를 지원하므로 가장 일반적으로 사용

### `name`과 `content` 속성

- `name` 속성은 메타데이터의 유형을 정의하고, `content` 속성은 해당 메타데이터의 값을 지정
- 주로 SEO와 관련된 정보를 전달하기 위해 사용
  - description: 검색 엔진 결과에 표시될 문서의 설명을 지정
  - keywords: 문서와 관련된 주요 키워드를 지정하여 검색 엔진 최적화를 도움
  - author: 문서의 작성자를 지정
  - viewport: 뷰포트 설정을 통해 반응형 웹 디자인을 구현

### `http-equiv`

- 웹 페이지가 브라우저에서 처리되는 방식을 제어할 때 사용
- content 속성에 명시된 값에 대한 HTTP 헤더를 제공
- http-equiv 속성이 명시되어 있다면, 반드시 content 속성도 함께 명시

### `Open Graph`

- 소셜 미디어에서 웹 페이지를 공유할 때 사용되는 메타데이터 정의
- `og:`로 시작하는 속성
  - `og:title`: 콘텐츠 제목
  - `og:description`: 콘텐츠 설명
  - `og:image`: 대표 이미지
  - `og:url`: 콘텐츠 URL

## 웹 표준이란?

- 웹에서 표준적으로 사용되는 기술을 의미
- 어떤 운영체제나 브라우저를 사용하더라도 웹 페이지가 동일하게 보여야 함
- `HTML`,`CSS`, `JavaScript` 주요 웹 기술 포함

### 웹 표준의 장점

1. 크로스 브라우징 호환성

   어떤 브라우저를 사용하더라도 웹 사이트가 동일하게 작동하므로 특정 브라우저에 의존하지 않고 사용자 경험을 일관성 있게 제공

2. 접근성 향상

시각장애인, 청각장애인 등 다양한 사용자가 웹 사이트를 이용할 수 있도록 HTML 구조 권장

3. 검색 엔진 최적화(SEO) 향상

웹 표준을 따르면 HTML 구조가 명확해지고, 검색 엔진이 사이트 콘텐츠를 효율적으로 분석할 수 있어 검색 노출에 유리

4. 유지 보수 향상

웹 표준을 준수하면 코드가 명료해지고 관리가 쉬워 유지 보수에 유리

## HTML5에서 추가된 내용이 있나요?

- **시맨틱** 태그 지원
  - (ex. `header`, `footer`, `section`, `article`, `nav` 등)
  - 접근성과 SEO에 유리
- 폼 입력 요소 확장
  - `date`, `email`, `number`, `range`, `color`, `tel` 등 추가
  - 다양한 입력 방식 지원으로 유효성 검증 및 모바일 기기 최적화 가능
- 멀티미디어 요소
  - 비디오를 쉽게 삽입할 수 있는 `<video>` 요소
  - `<audio>` 요소
- 그래픽 요소
  - `<canvas>` 자바스크립트를 통해 그림을 그릴 수 있음
  - `<svg>`는 벡터 그래픽을 위한 요소로 해상도에 영향을 받지 않아 고해상도 지원에 유리

## 웹 접근성이란?

모든 사람이 차별 없이 웹 사이트를 이용할 수 있도록 보장하는 권리이다.

## 웹 접근성에 맞는 마크업 예시 몇가지 말해보시오

1. 이미지에 대체 텍스트 사용

   이미지를 설명하는 alt 속성을 사용하여 이미지의 의미나 용도를 전달함으로써 시각 장애인도 이미지를 이해할 수 있게 돕는다.

   `<img alt="산타 모자를 쓰고 양 손에 선물 상자를 하나씩 든 여우 일러스트" />`

2. 시맨틱 태그 사용

   시맨틱 태그를 사용하여 사용자와 스크린 리더가 웹 페이지 구조를 쉽게 이해할 수 있도록 한다.

## 시멘틱 태그란 무엇인가 왜 사용하는가

시멘틱 태그는 `header`, `footer`, `section`, `article`, `nav`, `aside` 등 의미가 명확한 HTML 태그를 말하며, 콘텐츠의 구조와 의미를 보다 명확히 표현한다.(접근성 향상)

이를 통해 접근성을 향상시키고 브라우저와 검색 엔진이 HTML 문서의 구조를 쉽게 파악하고 SEO에 도움을 준다.

## 텍스트 관련 태그

1. `<h1> ~ <h6>`

   - 제목을 나타내는 태그
   - `<h1>` 이 가장 크게 표시되며 가장 중요한 제목이다.
   - 그 이후에 `<h2>~<h6>` 순서대로 사용되어야 한다.

2. `<p>`

   - 단락을 정의하는 태그
   - 문단의 텍스트를 묶어줌
   - 줄바꿈 ○

3. `<strong>`, `<em>`

   - 강조 태그
   - `<strong>`은 중요한 텍스트를 나타내고 굵게 표시된다. SEO에서도 중요
   - `<em>` 강조된 텍스트를 나타내며, 이탤릭체로 표시된다.
   - 줄바꿈 X

4. `<blockquote>`, `<q>`

   - 인용 태그
   - `<blockquote>`는 블록 형식의 긴 인용문을 나타내고 들여쓰기 형식으로 표시된다.
     - 줄바꿈 ○
   - `<q>`는 짧은 인용문에 사용되며 따옴표가 추가된다.
     - 줄바꿈 X

5. `<code>`, `<pre>`

   - 코드 태그
   - `<code>`는 짧은 코드 조각을 표시 줄바꿈 X
   - `<pre>`는 공백과 줄바꿈이 유지되는 코드 블록을 표시

6. `<br>`

   - 줄바꿈 태그

7. `<hr>`
   - 수평선 태그

## SEO란 무엇인가?

SEO는 검색 엔진 최적화로 사람들이 어떤 정보를 검색했을 때 웹 사이트가 상위에 노출되도록 하는 과정이다.

검색 엔진이 HTML 태그를 분석할 때, 의미 있는 시맨틱 태그를 사용하면 검색 엔진이 페이지를 더 잘 이해할 수 있어 도움이 된다.

이 과정을 통해 더 많은 방문자를 유입할 수 있다.

## Button 태그의 Default type은 무엇인가?

Submit 타입이다.

Button 태그가 form 내에서 사용될 때 type 속성을 지정하지 않으면 제출 기능이 작동하여 폼이 제출되고 페이지가 새로고침된다.

## Section 태그와 article 태그의 차이점

### `<section>`

- 문서의 주제를 구분짓는 데 사용
- 페이지의 구조를 정의하고 관련된 내용이 그룹화되어 있음을 나타냄

### `<article>`

- 독립적으로 배포하거나 재사용할 수 있는 콘텐츠의 조각

## 크로스 브라우징이란 무엇인가요?

크로스 브라우징은 다양한 브라우저에서 웹사이트가 동일하게 작동하도록 호환성을 맞추는 작업이다.

**왜 사용하는가?**

브라우저마다 HTML, CSS, JavaScript 등을 해석하는 방식이 조금씩 다르기 때문에, 특정 브라우저에서만 잘 보이거나 기능이 제대로 동작하는 문제가 생길 수 있기 때문

## 이미지 크기가 클 경우 렌더링 속도가 느려질텐데 이를 개선하기 위한 방법

- 이미지 용량 줄이기
  - 이미지 압축하거나 리사이징하여 로딩 속도 개선
- 이미지 형식 변경
  - svg 또는 webp을 사용하여 더 나은 압축률과 빠른 로딩 속도를 확보할 수 있음
- Lazy Loading
  - 화면에 처음 로드될 때 모든 이미지를 한 번에 로드하지 않고 사용자가 실제로 화면을 스크롤해서 이미지를 보려고 할 때 해당 이미지를 로드하는 것
  - 초기 로딩속도 향상
- CDN (Content Delivery Network)
  - CDN을 사용하여 이미지 파일을 사용자와 가까운 서버에서 전송받게 하면 로딩 속도 개선 가능
- 반응형 이미지
  - `<picture>` 태그나 `srcset` 속성을 사용하면 해상도에 맞는 이미지를 준비하면 디바이스에 따라 적합한 이미지가 로드됨
  - 불필요한 리소스 사용을 줄일 수 있다.

## SVG 장점과 단점

### SVG란?

- Scalable Vector Graphics로 XML 기반의 벡터 이미지 형식
- 크기 조절해도 해상도에 상관없이 이미지 품질이 떨어지지 않음

### 장점

- 이미지를 확대하거나 축소해도 품질이 유지되어 선명한 이미지를 제공할 수 있음
- 이미지 용량이 작아 로딩 속도가 빠르다.
- 애니메이션 및 상호작용이 가능하다.
- XML 텍스트로 구성되어 있어 검색 엔진이 읽을 수 있으므로 SEO에 도움이 됨

### 단점

- 색상과 세부 요소가 많은 복잡한 이미지일 경우 파일 크기가 커질 수 있어서 PNG, JPEG보다 비효율적일 수 있다.
- 최신 브라우저에서는 지원되지만 일부 구형 브라우저에서는 지원되지 않을 수 있음
- 복잡한 색상 표현에는 한계가 있어서 픽셀 이미지만큼의 세밀한 표현에는 적합하지 않다.

## SVG 내부 도형에 대해 아는게 있나요?

1. `<circle>`

   ```
   <circle cx="50" cy="50" r="40" />
   ```

   - 원을 그릴 때 사용
   - cx는 원의 중심 x 좌표, cy는 원의 중심 y 좌표, r은 원의 반지름

2. `<rect>`

   ```
   <rect x="10" y="10" width="80" height="80" fill="blue" rx="10" ry="10" />
   ```

   - 사각형을 그릴 때 사용
   - x, y: 사각형 시작 좌표
   - width: 너비
   - height: 높이
   - rx, ry: 둥근 모서리 반경

3. `<line>`

   ```
   <line x1="10" y1="10" x2="90" y2="90" />
   ```

   - 선을 그릴 때 사용
   - x1, y1: 시작점 좌표
   - x2, y2: 끝점 좌표

4. `<polyline>`

   ```
   <polyline points="10,90 50,10 90,90" />
   ```

   - 열린 꺾은선 그릴 때 사용
   - points: 각 점의 좌표
   - 각 포인트는 반드시 x 좌표와 y 좌표를 가지고 있어야 함

5. `<path>`
   ```
   <path d="M 20 230 Q 40 205, 50 230 T 90230" />
   ```
   - 복잡한 경로를 그릴 때 사용
   - d: 경로 데이터
   - M: 시작점, L: 선 긋기, C: 곡선

# display

- display는 요소를 어떻게 보여줄지 결정한다.

## block

- 요소가 한 줄을 모두 차지하고 다음 요소는 줄바꿈 후 배치된다.
- `width`, `height` 설정할 수 있음

## inline

- 요소가 한 줄을 모두 차지하지 않고 내용만큼의 너비만 차지한다.
- 줄바꿈 하지 않으며 `width`, `height` 설정해도 적용 안됨

## inline-block

- inline과 같이 한 줄에 여러 요소가 배치될 수 있다.
- inline와 다르게 `width`, `height` 값을 설정할 수 있다.
- 즉, 요소가 차지하는 영역은 내용 크기에 맞춰지지만, 크기 조정이 가능하다.

## none

- 요소를 화면에서 숨긴다.
- 공간을 차지하지 않고 숨김처리 함. 즉, DOM에서 제거된 것처럼 동작한다.
- `visibility: hidden`과 다르다.

## position에 대하여 설명해보세요. 🔥

- position은 HTML 문서 상에서 요소가 배치되는 방식을 결정한다.

## static

- `position` 속성의 기본값으로 배치 기준이 없다.
- 왼쪽에서 오른쪽, 위에서 아래로 배치된다.
- `top`, `right`, `bottom`, `left`와 같은 다른 위치 지정 속성을 사용할 수 없음

## relative

- 자기 자신의 원래 위치 기준으로 상대적으로 이동한다.
- 다른 요소와 겹치지 않고 여전히 공간을 차지하기 때문에 레이아웃 흐름에 영향을 주지 않는다.

## fixed

- 웹 브라우저 화면 전체 기준으로 배치한다.
- 스크롤을 해도 화면에 항상 고정된 위치에 있다.

## absolute

- 가장 가까운 부모 요소를 기준으로 위치가 설정된다.
- 단, 부모 요소의 position이 `relative`, `absolute`, `fixed` 일 때만 기준이 된다.
- 부모 요소가 없다면 초기 컨테이닝 블록을 기준(보통 `body` 태그)으로 배치된다.
- 요소를 일반적인 문서 흐름에서 제거하기 때문에 다른 요소들과 겹쳐지거나 특정 위치에 고정될 수 있다.

## float가 어떻게 작동하는가

- `float`는 웹 디자인에서 요소를 화면의 왼쪽이나 오른쪽으로 띄워, 다른 콘텐츠가 그 주위를 감싸도록 만드는 속성이다. 주로 이미지나 박스를 텍스트 옆에 배치할 때 사용된다.
- `float`가 적용된 요소는 문서 흐름에서 벗어나지만, 부모 요소 내에서만 떠 있으며, 텍스트나 다른 요소들이 그 주위를 감싸게 된다.
- `float`된 요소는 블록 요소로 취급되며, 그 크기만큼 다른 콘텐츠들이 옆으로 밀려난다.
- `float`로 떠 있는 요소가 다른 요소에 영향을 미치지 않게 하려면, `clear` 속성을 사용하여 그 아래에 있는 요소가 `float` 요소와 겹치지 않도록 할 수 있다.

### 주의할 점

- `float` 속성을 사용하면 부모 요소가 자식 요소의 높이를 포함하지 않게 될 수 있다.
- 이를 해결하려면 부모 요소에 `clearfix` 기법을 사용하거나, `overflow: hidden`을 설정하여 자식 요소가 `float`로 떠 있는 경우에도 부모의 높이를 계산하게 할 수 있다.

# Flexbox나 Grid 스펙을 사용해본 적이 있나요 ?
## flex 를 사용하는 이유가 무엇인가요?
`flex`는 1차원 레이아웃으로 주로 요소를 가로 또는 세로로 정렬하고 배치하는 데 사용된다.

- 간단한 정렬과 배치
   - 요소 간의 간격 조절과 중앙 정렬을 더 효율적으로 해줌
- 반응형 디자인과 유동적인 크기 조절
   - 화면 크기에 따라 요소들이 자동으로 배치되고, 요소의 크기도 유연하게 변경되어 다양한 화면 크기에 맞게 조정된다.


## Grid를 사용하는 이유가 무엇인가요?
`grid`는 2차원 레이아웃으로 가로와 세로의 행과 열을 조합하여 복잡한 레이아웃을 구성하는 데 적합하다.

- 2차원 레이아웃
   - 행과 열을 모두 활용해 복잡한 레이아웃을 쉽게 구현할 수 있다.
- 반응형 디자인과 자동 조정
   - 브라우저 창 크기에 따라 레이아웃과 요소의 크기가 자동으로 조정된다.

## 이미지 태그를 스타일로 대체하는 법
이미지 태그 대신 `<div>`와 같은 요소에 CSS로 `background-image`를 설정하여 이미지를 배경으로 추가한다.

이 방법은 코드가 간단해지고 CSS로 이미지 크기 조정, 위치 설정 등을 할 수 있지만, HTML의 `<img>` 태그가 아니라서 검색 엔진이 이미지를 인식하지 못할 수 있다.
